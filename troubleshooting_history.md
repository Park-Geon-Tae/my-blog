# 유튜브 블로그 자동화 스크립트(youtube_to_blog.py) 트러블슈팅 이력

본 문서는 `youtube_to_blog.py` 구축 및 고도화 과정에서 발생한 주요 문제점들과 그 해결 과정을 요약한 것입니다.

## 1. 블로그 포스팅 품질 및 톤앤매너 문제
* **문제점:** 초기 스크립트가 유튜브 원문의 비유나 농담, 구어체를 그대로 복사하여 전문성이 떨어지고 글이 가벼워지는 문제 발생.
* **해결 방법:** 
  * 시스템 프롬프트를 '9년 차 베테랑 PLC/전기설계 엔지니어 Mr.FIX' 페르소나로 전면 개편.
  * 감정적 서술을 100% 배제하고 객관적 팩트와 핵심 데이터만 추출하도록 지침 강화.
  * 가독성을 위한 목차(Table of Contents) 생성, 명확한 결론(Conclusion), 다음 포스팅 예고(Teaser) 및 고정 클로징 멘트 조건을 프롬프트에 추가하여 리포트 형태의 완결성을 확보.

## 2. Gemini API 응답 텍스트 잘림 현상
* **문제점:** 생성된 마크다운 텍스트 콘텐츠가 끝까지 작성되지 못하고 중간에 뚝 끊기는 현상 발생.
* **해결 방법:** Gemini API의 `GenerateContentConfig` 설정에 `max_output_tokens=8192` 파라미터를 명시적으로 추가하여 출력 토큰 제한을 대폭 늘려 해결.

## 3. IDE(에디터) 타입 체킹 및 Lint 에러
* **문제점:** 코드 실행에는 문제가 없으나, VSCode 등 에디터의 엄격한 타입 체커(Pyre)가 `None` 가능성을 오인식하여 수많은 빨간 줄(에러)을 표시하고 가상환경 모듈의 import 경로를 찾지 못함.
* **해결 방법:** 
  * 모든 주요 함수에 Python 3.10+ 타입 힌팅(`: str`, `-> dict | None`) 적용.
  * `assert transcript is not None` 등 논리적 타입 좁히기(Type Narrowing) 추가.
  * 인터프리터 환경 불일치로 인한 거짓 경고와 슬라이싱 오인식 부분에 `# type: ignore` 주석을 추가하여 에디터 환경을 깔끔하게 유지.

## 4. 이미지 생성 API 결제 제약 (400 Error)
* **문제점:** 스크립트가 초기에는 Gemini의 `Imagen 3` 모델을 호출했으나, 무료 API 키 환경이기 때문에 서버에서 `400 INVALID_ARGUMENT (Paid plans only)` 에러를 반환. 이로 인해 빈 폴더만 생성되고 마크다운에 엑스박스가 뜸.
* **해결 방법:** 
  * 에러를 묵인하지 않고 터미널에 상세한 Traceback과 절대 경로, `os.path.exists()` 검증 로그를 띄우도록 에러 핸들링 보강하여 원인 규명.

## 5. 대체 무료 이미지 API(Pollinations)의 봇 차단 (403 / HTML 반환)
* **문제점:** Imagen 3의 대체재로 무료 도메인인 `pollinations.ai`를 연동했으나, 파이썬 기본 `urllib` 접근 시 Cloudflare 보안에 막혀 403 에러 발생. `User-Agent`를 브라우저로 위장하여 우회했더니, 서버가 봇 차단을 위해 실제 이미지가 아닌 웹 앱의 HTML(방어 페이지)을 다운로드시켰고 이로 인해 Astro 빌드 서버가 크래시남.
* **해결 방법 (현재 최종 적용):**
  * 프롬프트를 전체 문장이 아닌 '1~2개의 영문 핵심 키워드'만 추출하도록 간소화.
  * 주제와 연관된 이미지를 즉시 반환하는 `loremflickr.com` API(`https://loremflickr.com/1024/576/{keyword}`)로 엔드포인트를 교체하여 보안 차단을 우회하고 안정적인 썸네일 수급 자동화를 완성.
