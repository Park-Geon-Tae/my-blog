# YouTube SEO 블로그 포스팅 자동화 에이전트 상세 구조도

## 1. 개요 (Overview)
- **에이전트 역할:** 입력된 유튜브 영상을 분석하여 전문적인 IT/기술 블로그 포스팅(마크다운 형식)을 자동으로 작성하는 지능형 스크립트.
- **개발 환경:** Python 기반
- **주요 기반 기술:** Gemini API (Free Tier - Gemini 1.5 Pro)

---

## 2. 자동화 파이프라인 단계 (Workflow Steps)

### 1단계: 유튜브 URL 입력 (Input)
- 사용자가 스크립트 실행 시 대상 유튜브 영상의 URL을 입력.
- 영상 메타데이터(제목, 채널명, 게시일 등) 초기 수집.

### 2단계: 자막 추출 (Subtitle Extraction)
- `youtube-transcript-api` 등의 라이브러리를 활용하여 영상의 전체 자막(텍스트) 추출.
- 자막이 없는 경우 음성 인식(Whisper API 등) 대안 마련 (선택 사항).
- 튜브 자막 추출 시 유료 API 대신 yt-dlp와 같은 무료 도구를 사용하여 비용을 절감하는 방안

### 3단계: Gemini 1.5 Pro 분석 (Analysis)
- 추출된 대량의 텍스트 데이터를 Gemini 1.5 Pro(무료 티어 롱컨텍스트 활용) 모델에 전달.
- 영상 핵심 주제, 주요 인사이트, 기술적 내용 및 요약 정보 분석.

### 4단계: SEO 최적화 초안 작성 (SEO Optimization & Drafting)
- 분석된 내용을 바탕으로 검색 엔진(SEO)에 최적화된 블로그 포스팅 초안 작성.
- 관련 검색어(키워드) 도출 및 본문 내 자연스러운 배치.
- 텍스트 가독성을 높이는 단락 분리 및 블릿 포인트 활용.

### 5단계: AI 이미지 프롬프트 생성 (Prompt Generation)
- 작성된 블로그 포스팅 내용을 기반으로, 본문 삽입에 적합한 2장의 이미지를 기획.
- 기획된 컨셉을 바탕으로 이미지 생성 AI(DALL-E, Midjourney, Stable Diffusion 등)에 전달할 영문 최적화 프롬프트 자동 생성.

### 6단계: 이미지 생성 (Image Generation)
- 프롬프트를 사용하여 AI 이미지 생성 API를 호출하고 총 2장의 이미지를 취득.
- 블로그 에셋 폴더(`src/assets/images/`)에 이미지 자동 다운로드 및 최적화 저장.

### 7단계: 최종 마크다운 저장 (Export)
- 블로그 테마(AstroPaper)에 맞는 Frontmatter(`title`, `description`, `pubDatetime`, `tags` 등) 형식 적용.
- 작성된 텍스트와 다운로드된 2개의 이미지 상대 경로 링크를 결합하여 최종 `.md` 마크다운 파일 조립.
- `src/content/blog/` (또는 지정 경로)에 마크다운 파일로 저장 완료.

---

## 3. 출력 규격 (Output Specifications)

### 1. 후킹 문구 (Hooking Intro)
- 포스팅의 최상단 첫 줄은 독자의 이목을 단숨에 사로잡을 수 있는 매력적인 **후킹 문구(150자 내외)** 로 시작.
- (예: "매일 반복되는 지루한 작업에 지치셨나요? 오늘은 단 10분 투자로 100시간의 여유를 만들어줄 파이썬 자동화 스크립트 작성법을 공개합니다.")

### 2. 헤딩 구조 (Heading Structure)
- SEO 로직 및 가독성 향상을 위해 **H1(제목), H2(대주제), H3(소주제)** 구조를 엄격하게 준수.
- `<h1>` (Frontmatter title로 대체) -> `## H2 헤딩` -> `### H3 헤딩` 계층 구조 유지.

### 3. AI 이미지 포함 (Images)
- 생성된 AI 이미지 2장을 포스팅 본문의 흐름에 맞는 적절한 위치에 배치(마크다운 내 이미지 문법 `![alt text](path)` 적용).
- 썸네일(Hero Image) 용도로 1장, 본문 이해를 돕는 용도로 1장 등 전략적 배치.
